232  <p align="justify"> Developed through a collaborative effort of the National Institute for Computational Sciences (NICS) and Texas Advanced Computing Center (TACC), XALT is a mechanism for following users' jobs and environments on a computer cluster. It provides a census of libraries and applications and automatically filters user issues, yielding exactly the type of job tracking information that most computing centers need or want.
233  </p>
234  <p align="justify"> XALT reveals information such as the number of users and projects associated with a particular library or execuatable; whether a library maintained by a center is applied (and how often); and whether center-provided packages are used more, or less, than user-installed packages. XALT enables computing centers to notify users, after the fact, that a library was buggy; provide information on how an execuatable was build; catch compile time/run time differences; and indentify applications that are using deprecated libraries or just identify old binaries. After an alpha release in summer 2014 with many aplha testers, as well as a beta release on November 10, 2014, XALT is now in production use on Stampede, Darter, and Keeneland supercomputers. It has been ported to the Nautilus supercomputer but it is not yet in production. XALT is a new generation of user environment tracking and problem detection based on the Automatic Library Tracking Database (NICS) and Lariat (TACC) prototypes.
235  </p>
